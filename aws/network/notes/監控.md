🕵️‍♂️ 什麼是 IDS 與 IPS？
它們是網路安全設備的兩大護法，通常會放在 VPC 的出入口（像是防火牆的進階版）。

IDS (Intrusion Detection System / 入侵偵測系統)：
就像是 「監視攝影機」。它會側錄經過的封包，如果發現有駭客在做壞事（例如 SQL 隱碼攻擊），它會「發出警報 (Alert)」，但它不會主動阻擋封包。

IPS (Intrusion Prevention System / 入侵防禦系統)：
就像是 「配槍的警衛」。它不僅會看，一旦發現壞人，它會直接把封包 「丟棄 / 阻擋 (Block / Filter)」。

😈 什麼是 Promiscuous Mode (混雜模式)？
在傳統的地端機房，Switch（交換器）上面有很多台電腦。正常的網卡只會接收「指名給自己」的封包。
如果你把網卡開成 「混雜模式 (Promiscuous Mode)」，這張網卡就會變成「竊聽狂」，它可以聽到同一個 Switch 上所有其他人在傳遞的封包。傳統的 IDS 就是靠這種模式來監控全公司的流量。

📜 什麼是 CloudTrail？
AWS 的 「行車紀錄器 / 稽核員」。不管你是透過網頁按鈕，還是下指令，只要你對 AWS 環境做了「任何設定上的改變」（例如：開了一台機器、改了安全組、砍了資料庫），這叫做 API Changes，CloudTrail 會把「誰、在什麼時候、做了什麼事」全部記錄下來。

2. 【情境掃描】（Scenario Overview）
題目說：金融法規要求你必須監控兩件事：

"all changes in an environment" (環境中所有的設定變更)。

"all traffic sent to and from" (進出環境的所有網路流量)。

3. 【選項過濾：AWS 的絕對紅線】
這題的核心考點，其實只在考你知不知道 AWS 底層網路的一條鐵律：

⛔ AWS 的絕對禁忌：VPC 內「嚴禁」使用混雜模式 (Promiscuous Mode)！

在 AWS 虛擬化平台中，為了保護租戶的安全，底層的 Hypervisor 會強制隔離流量。你的網卡絕對不可能竊聽到別人的流量。任何試圖開啟混雜模式的操作，都會被 AWS 底層直接丟棄。

帶著這個鐵律，我們來看選項：

❌ A. Configure an IPS/IDS in promiscuous mode...

死因： 關鍵字 promiscuous mode。在 AWS 裡這招行不通，直接淘汰。

❌ B. Configure an IPS/IDS system... using promiscuous mode...

死因： 一樣踩到了 promiscuous mode 的地雷，淘汰。

剩下來的，閉著眼睛也只能選 C 和 D 了！我們來驗證一下：

🎯 C. Configure an IPS/IDS to listen and block... Configure CloudTrail...

為什麼對： 這是最完美的一段話。前半段用 IPS 監控/阻擋流量；後半段用 CloudTrail 結合 CloudWatch Logs 來監控環境設定的變更 (API changes)。完美呼應了題目的兩個要求。

🎯 D. Configure an IPS/IDS system... that monitors, filters, and alerts...

為什麼對： 這是標準的 IPS 運作方式（監控、過濾、警報），而且沒有提到違法的混雜模式。雖然它只回答了題目一半的需求（流量），但在「四選二」的機制下，它是唯一合法的第二個選項。
