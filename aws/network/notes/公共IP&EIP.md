這題是 AWS 考試中非常容易讓新手混淆的「文字陷阱」，重點在於區分 Public IP (一般公有 IP) 與 Elastic IP (EIP, 彈性 IP) 的生命週期。正確答案（即描述錯誤的選項）是 D. You can manually associate or disassociate a public IP address.🔍 題幹手術：Public IP vs. Elastic IP在 AWS 的世界裡，並不是所有會對外的 IP 都叫一樣的名字。它們的控制權完全不同：1. 一般公有 IP (Public IP) —— 「隨機分配的臨時工」特性： 當你在公有子網啟動 EC2 時，AWS 會自動從它的資源池抓一個給你。致命點： 你**「不能」手動把它拔掉換給別人，也「不能」**手動拿一個特定的 Public IP 塞給某台機器。消失機制： 只要機器 Stop 或 Terminate，這個 IP 就會被收回池子裡（選項 B 是對的）。當你再次 Start 機器時，你會得到一個全新、不同的 IP。2. 彈性 IP (Elastic IP) —— 「你專屬的固定資產」特性： 這是你主動跟 AWS 申請的固定 IP。控制權： 你**「可以」**隨時手動把這個 IP 指向機器 A，下一秒再手動拆下來指向機器 B。這才叫 "Manually associate / disassociate"。💡 選項細節拆解A. Assigned from Amazon's pool (描述正確): 沒錯，AWS 有一堆公有 IP 池，機器開機就分一個給你。B. Disassociated, added back to the pool (描述正確): 當機器關機或刪除，這個臨時 IP 就回歸池子給別人用了。C. Reachable from the internet (描述正確): 這是公有 IP 的唯一存在意義。D. Manually associate... (描述錯誤 ❌): * 只有 Elastic IP 才能讓你「手動操作」。一般 Public IP 是跟著機器的開關自動「附著」或「脫落」的，你沒有手動干預的權限。🚀 專家教練的「考場反射動作」當你看到 "Public IP" (沒加 Elastic 這個字) 時：動作能不能做？機器停止後 IP 會變嗎？✅ 會變我能手動幫機器換 IP 嗎？❌ 不能 (那是 EIP 的特權)我能自己選號碼嗎？❌ 不能一句話記住： > Public IP 是 AWS 給你的「租屋處隨機門牌」（退租就沒了且不能換）；Elastic IP 才是你「買下來的永久門牌」（你可以帶著它搬家）。
