## 1. Hybrid Networking 核心角色

### `VGW`

- `VGW = Virtual Private Gateway`
- 它是單一 VPC 的傳統邊界閘道
- 常拿來做：
  - `Site-to-Site VPN`
  - `Private VIF` 終止點

### `TGW`

- `TGW = Transit Gateway`
- 它不是單一 VPC 邊界，而是 many VPC / many networks 的中央 hub
- 適合：
  - 多 VPC 互聯
  - 多帳號
  - hub-and-spoke
  - hybrid transit

### `DXGW`

- `DXGW = Direct Connect Gateway`
- 它是 DX 的邏輯中介站，讓一條 DX 不用只綁死在單一 VPC
- 常見價值：
  - DX 跨 Region
  - 多 VPC / 多帳號擴展
  - 對接 `VGW` 或 `TGW`

### 今天建立的超短口訣

- `VGW = 單 VPC 邊界`
- `TGW = 多網路中樞`
- `DXGW = DX 的擴展器`

## 2. BGP 三個必背屬性

### `LOCAL_PREF`

- 主要控制 **outbound path**
- 也就是「我自己出去想走哪條」

### `AS_PATH prepend`

- 常拿來影響 **inbound path**
- 也就是讓對方覺得某條路比較遠、比較不想從那裡回來

### `MED`

- 是對鄰居的偏好提示
- 常見於多條連線時的相對偏好
- 不是萬能控制鈕，考題常故意拿它跟 `AS_PATH prepend` 混淆

### 今天的結論

- `LOCAL_PREF = 出站偏好`
- `AS_PATH prepend = 入站偏好常用招`
- `MED = 提示，不是保證`

## 3. AWS 路由優先序

### 必背順序

`Longest Prefix -> Static -> DX -> VPN static -> VPN BGP`

### 今天真正理解到的重點

- `Longest prefix` 永遠最強
- 手動 `static route` 會壓過 propagated route
- 同 prefix 下，`DX` 預設優先於 `VPN`
- 如果想讓某個特定網段改走另一條路，第一反應通常應該是：
  - 「我能不能宣告更具體的 prefix？」

### 例子

- DX 廣告 `172.16.0.0/16`
- VPN 廣告 `172.16.10.0/24`
- 流量到 `172.16.10.8`

結論：
- 走 `VPN`
- 原因不是 VPN 本身比較強，而是 `/24` 比 `/16` 更具體

## 4. Hybrid 題型演練

### 題型 1：特定 CIDR 改走 VPN

**情境：**
- 同時有 DX 和 VPN
- 平常希望都走 DX
- 但 `10.20.30.0/24` 要改走 VPN

**正解：**
- 在 VPN 端宣告更具體的 prefix

**考點：**
- 先看 `Longest prefix`
- 不要一開始就先跳去想 `LOCAL_PREF`

### 題型 2：大量 VPC、多帳號、多 Region、單組 DX 要擴展

**情境：**
- 40 個 VPC
- 多帳號
- 多 Region
- 想用一組 DX 支援整體架構

**正解：**
- `Transit VIF -> DXGW -> TGW`

**考點：**
- 這是在考 scalable transit hub
- 不是 per-VPC private VIF 的傳統做法

### 題型 3：兩個 VPC 掛同一個 DXGW 是否互通

**正解：**
- 不可以

**考點：**
- `DXGW` 不是 `TGW`
- `DXGW` 不提供 VPC-to-VPC transitive routing

## 5. 今天的易混點與結論

### 易混點 1

- 容易把 `DXGW` 當成 transit service
- 但真正提供 many-to-many transit routing 的是 `TGW`

### 易混點 2

- 容易以為選路都靠 BGP attribute
- 但很多題真正先決的是：
  - `Longest prefix`
  - `Static route`

### 易混點 3

- `VGW`、`TGW`、`DXGW` 都像 gateway，很容易混
- 但它們分別在回答不同層次的問題：
  - `VGW`：一個 VPC 的邊界
  - `TGW`：很多網路的 hub
  - `DXGW`：DX 怎麼優雅擴展
