# AWS 網路解題紀錄 - 2026-04-10

## Q001 監控到達 EC2 的流量

- 主題分類：`VPC`
- 題幹關鍵字：`monitor the traffic that is reaching your EC2 instances`
- 正解：`VPC Flow Logs`
- 我原本容易想到：`VPC Reachability Analyzer`
- 錯因：`服務不熟`
- 正確判斷依據：題目問的是「監控或記錄實際到達 EC2 的流量」，這是 `VPC Flow Logs`；`Reachability Analyzer` 是做靜態路徑可達性分析，不是流量監控。
- 錯誤選項分析：
  - `Security Groups`：它是 allow 規則集合，不提供流量監控或紀錄功能。
  - `NACLs`：它是 stateless subnet 邊界規則，不是觀測工具。
  - `Subnet Flow Logs`：正式概念仍然是 `VPC Flow Logs` 套用在不同層級，這種寫法常是干擾考你服務名稱。
- 陷阱：
  - `Reachability Analyzer` 很像「看流量」，但它其實是看路徑是否可達。
  - 題目只要出現 `monitor traffic`、`log traffic`，優先想 `Flow Logs`。
  - `Flow Logs` 看的是 metadata，不是 payload。
- 一句口訣：`看實際流量 -> Flow Logs；看路通不通 -> Reachability Analyzer`

## Q002 DHCP Options Set 哪個敘述是錯的

- 主題分類：`VPC`
- 題幹關鍵字：`Which of the following is false when it comes to DHCP Options Set`
- 正解：`You can modify the DHCP options set after they have been created.`
- 錯因：`服務不熟`
- 正確判斷依據：`DHCP options set` 建立後不能直接修改；若要改內容，要建立新的 options set 再重新關聯到 VPC。
- 錯誤選項分析：
  - `You can have multiple sets of DHCP options`：這是對的，帳號裡可以建立多個 options set。
  - `One DHCP options can be associated with one VPC at a time`：這句英文很爛，但真正考點通常是「一個 VPC 同一時間只能掛一個 options set」。
  - `If you delete a VPC, the DHCP options set associated with the VPC are also deleted`：這種敘述在考題平台上常常寫得不嚴謹，要特別小心不要比 A 更先被它誤導。
- 陷阱：
  - 最穩的鐵則是 `DHCP options set 不能直接修改`。
  - 出題者很愛拿「association 關係」和「resource lifecycle」混在一起考。
  - 如果題目延伸到 Hybrid DNS，很多時候更好的答案其實是 `Route 53 Resolver`，不是去改 DHCP。
- 一句口訣：`DHCP options set 不能改，只能重建再換掛載`

## Q003 Private subnet instance 如何對外上網

- 主題分類：`VPC`
- 題幹關鍵字：`instances in the private subnet can communicate with the Internet`
- 正解：`NAT Instances`、`NAT Gateway`
- 我原本容易選：`Internet Gateway`
- 錯因：`題幹關鍵字漏看`
- 正確判斷依據：題目在考「private subnet instance 對外上網時，用哪種 NAT 出口設備」，答案是 `NAT Instance` 或 `NAT Gateway`。`Internet Gateway` 是整個 VPC 對 Internet 的出口元件，但 private subnet 的 instance 不能直接靠它上網。
- 錯誤選項分析：
  - `Internet gateway`：它是 VPC 的 Internet 出口，但 private subnet instance 不能直接用它出網。
  - `AWS Direct Connect`：它是 private/hybrid 連線方案，不是拿來讓 private subnet 存取 public Internet 的標準答案。
- 陷阱：
  - 題目如果問完整路徑，你會想到 `NAT + IGW`，這沒有錯。
  - 但只要題目在選 NAT 類型或 private subnet 出網方案，答案通常會縮到 `NAT Gateway / NAT Instance`。
  - `private subnet` 看到就先排除「直接走 IGW」。
- 一句口訣：`Private subnet 要上網 -> 想 NAT；Public subnet 要上網 -> 想 IGW`

## Q004 哪些服務可用來緩解 DDoS

- 主題分類：`Security`
- 題幹關鍵字：`services can be used to mitigate DDoS attacks`
- 正解：`Route 53`、`CloudFront`、`Elastic Load Balancer`
- 錯因：`無，答對`
- 正確判斷依據：這題在考 AWS 常見的 DDoS 緩解前線服務組合。`Route 53`、`CloudFront`、`ELB` 都能協助吸收、分散或承接大流量攻擊。
- 錯誤選項分析：
  - `SOS`：不是 AWS 這類題型的標準服務名稱，屬於明顯干擾選項。
- 陷阱：
  - 如果選項裡有 `AWS Shield`，不要忘了它才是專門的 DDoS 防護品牌。
  - 但題目若是問「哪些服務可用來協助 mitigate」，`Route 53 + CloudFront + ELB` 也很常是標準答案組合。
  - 出題者常故意把「DDoS 專門產品」和「具備緩解能力的前線服務」混在一起。
- 一句口訣：`DDoS 前線三件組 -> Route 53 + CloudFront + ELB`

## Q005 多台 web server 要平均分流

- 主題分類：`Load Balancing`
- 題幹關鍵字：`traffic to an application is evenly balanced`、`multiple web servers`
- 正解：`Place all your web servers behind ELB. Configure a Route53 CNAME to point to the ELB DNS name.`
- 錯因：`無，答對`
- 正確判斷依據：題目明確在考「多台 web server 的流量平均分配」，標準答案就是 `ELB`。`NAT` 是出網用途，`CloudFront` 是 CDN，不是最直接的負載平衡答案。
- 錯誤選項分析：
  - `NAT instance`：用途是讓 private subnet 出網，不負責把 inbound 請求平均分配到多台 web server。
  - `CloudFront distribution`：它是 CDN，不是這題最直接的 load balancing 解法，而且把 origin 指到多台私網 IP 的寫法也很不標準。
  - `ELB with an EIP`：這不是這類基礎題的標準搭法，出題者在用 `EIP` 這個熟詞混淆你。
- 陷阱：
  - 題幹只要出現 `multiple web servers`、`evenly balanced`，先想 `ELB`。
  - `CloudFront` 很容易因為也在前面接流量而被誤選，但它的主要角色是 CDN / edge cache。
  - 考試若寫 `Route 53 CNAME` 不一定完全嚴謹，但不影響你判斷 `ELB` 才是考點。
- 一句口訣：`多台 web server 平均分流 -> ELB`
