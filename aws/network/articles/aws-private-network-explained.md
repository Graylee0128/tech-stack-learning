# AWS 的「內網」到底是什麼？我用搞懂 PrivateLink vs DX SiteLink 的過程來解釋

## 文章骨架

### 【第一段：拋出困惑】
學 AWS 的時候我發現一件事——
AWS 有很多東西都在講「不走公網」，
但 PrivateLink、Direct Connect、SiteLink
這三個到底差在哪？我一直沒想清楚。

---

### 【第二段：我的第一個誤解】

**初始誤解**：我以為 PrivateLink = 私有連線 = 就是內網

**實際上的區別**：
- **PrivateLink** → 服務之間的連接方式（VPC 內 or VPC 間）
- **Direct Connect (DX)** → 物理專線（您的資料中心 ↔ AWS）
- **Site-to-Site VPN (SiteLink)** → 網路層加密隧道（您的網路 ↔ VPC）

**為什麼會搞混**：
- 都是「不走公網」
- 但各自解決不同層次的問題
- 應用場景完全不一樣

---

### 【第三段：釐清的關鍵轉折點】

直到我把這三個東西畫成一張圖，才發現......

**關鍵洞察（待深化）**：
```
層級關係：
- PrivateLink：VPC 內 or AWS 生態內的「服務發現 + 安全連接」
- Site-to-Site VPN：您的網路 → VPC 的「加密隧道」
- Direct Connect：物理基礎設施層的「專用線路」
```

**它們的共同點**：都避免流量經過公網
**它們的差異**：
1. 連接的雙方不同（VPC←→VPC / 本地←→VPC / 本地←→AWS）
2. 實現層次不同（應用層 / 傳輸層 / 物理層）
3. 性能、成本、複雜度差異巨大

---

### 【結尾：留下懸念】

這篇是我釐清過程的第一步，我發現只要搞清楚「它們各自連接的是誰」和「在哪一層解決問題」，就能理解它們為什麼長這樣。

下一篇我會把那張圖畫出來，並加上實際使用場景。
