# 內網是啥：從家用寬頻到 AWS Direct Connect

## 第一層：基礎概念

### LAN vs WAN

| 特性 | LAN（區域網） | WAN（廣域網） |
|------|---|---|
| **範圍** | 同一棟樓或地點 | 跨地點 |
| **例子** | 公司內網、家裡 WiFi | 電信商連線 |
| **管理** | 自己管 | 電信商管 |
| **費用** | 自己承擔 | 月租費 |

**LAN ≈ VPC**：LAN 是實體網路，VPC 是 AWS 上的虛擬私有網路，功能概念相似。

### 公網 vs 專線 vs AWS 骨幹

| 類型 | 走哪裡 | 共用？ | 例子 |
|------|---|---|---|
| **公網（Internet）** | 各大 ISP 互連的全球網路 | 是 | 家用寬頻、公司寬頻 |
| **電信商專線（MPLS）** | 電信商自己的私有骨幹 | 否 | 企業向中華電信租的專線 |
| **AWS 骨幹** | AWS 自己的海底電纜+光纖 | 否 | DX 進去之後走的路 |

## 第二層：家用/公司寬頻的封包路徑

### 封包走向

```
你的設備
   ↓ LAN
WiFi Router（例如 Totolink）
   ↓ WAN 口
大樓匯聚設備（社區寬頻）或直接光纖
   ↓
中華電信地區機房（BRAS）
  → PPPoE 認證、NAT、給你公網 IP
   ↓ ← 從這裡開始是公網
中華電信骨幹 → TWIX（台灣網路交換中心）
   ↓ BGP 動態路由
對方 ISP → 目標伺服器
```

### 為什麼晚上會變慢？

**共用發生在三層**：
1. 大樓設備
2. 電信商地區機房
3. 電信商對外出口

→ 晚上尖峰時刻超賣，所以變慢

### 快速判斷

**看到 Totolik → 寬頻 → 走公網** 🌐

## 第三層：Direct Connect（DX）是什麼

### DX 不是什麼

❌ AWS 不會幫你拉線到辦公室

### DX 是什麼

✅ AWS 在指定機房（DX Location）開了一個入口
✅ 你要自己想辦法把線牽到那個入口

### 實體路徑

```
你的辦公室
   ↓ 你租的電信商專線（WAN，不走公網）
DX Location（例如台北 Equinix 機房）
  → Colocation 機房
  → AWS 在這裡放了自己的 DX Router
  → 你或電信商拉 cross-connect 光纖接進 AWS 設備
   ↓ 從這裡開始是 AWS 私有骨幹
AWS DX Edge Router
   ↓
AWS 骨幹網路（海底電纜+光纖，私有）
   ↓
目標 AWS Region / VPC / 服務
```

### 關鍵概念

#### Colocation（機房租賃）

- 你把自己的硬體設備放在別人的機房
- 機房提供：電力、冷氣、安全
- 設備管理：還是你自己管
- **類比**：租停車位，車還是你的

#### Cross-connect（機房內短光纖）

- 在同一棟 colocation 機房裡
- 你的設備 ↔ AWS 設備 之間的短光纖（幾公尺）

### 兩種接入方式

| 方式 | 做法 | 優點 | 缺點 |
|------|------|------|------|
| **自己接** | 自己去 DX Location 租機架、拉 cross-connect | 自主性高 | 需要自己管理硬體 |
| **DX Partner** | 找 DX Partner（中華電信、遠傳等）購買接入服務 | 免去自己租機架、他們已接好 AWS | 依賴合作商 |

## 第四層：進入 AWS 之後的路由

### VGW vs TGW

| 設備 | 功能 | 連接對象 | 適用場景 |
|------|------|---------|---------|
| **VGW**<br/>Virtual Private Gateway | AWS 端的 VPN/DX 接入點 | 掛在單一 VPC 上 | 單個 VPC 需要 DX 連線 |
| **TGW**<br/>Transit Gateway | AWS 的中央 Hub Router | 多個 VPC + 多條 DX/VPN | 複雜企業網路拓撲 |

### 🔒 關鍵認知：完全私有

VGW 和 TGW 都住在 AWS 私有骨幹裡
→ 封包經過時**完全不碰公網**

### 深化：Direct Connect 的拓撲架構演進

當企業有 **多個 VPC** 要透過 DX 連接時，架構設計會面臨重大取捨。

#### ❌ 初級方案：Per-VPC Private VIF（線性擴展）

**拓撲**：
```
On-prem Router
  └─ Direct Connect (physical)
      ├─ Private VIF (VLAN 101) ─ BGP ─ VGW ─ VPC-A
      ├─ Private VIF (VLAN 102) ─ BGP ─ VGW ─ VPC-B
      ├─ Private VIF (VLAN 103) ─ BGP ─ VGW ─ VPC-C
      └─ ...（每加一個 VPC，就加一個 Private VIF + 一個 VLAN + 一組 BGP）
```

**痛點**：
- **線性擴展**：VPC 數量 ↑ ⇒ VIF/VLAN/BGP session 全部 ↑
- **容易撞 Quota**：VIF 上限、VLAN 數量限制、路由表關聯上限
- **管理成本高**：每個 VPC 都是獨立接線、獨立控管、獨立 BGP 會話

#### ✅ 推薦方案：Transit VIF + DXGW + TGW Hub（Hub-and-Spoke）

**拓撲**：
```
On-prem Router
  └─ Direct Connect (physical)
      └─ Transit VIF (single VLAN) ─ BGP ─ Direct Connect Gateway (DXGW)
                                   └─ Associate ─ Transit Gateway (TGW Hub)
                                                    ├─ TGW Attachment ─ VPC-A
                                                    ├─ TGW Attachment ─ VPC-B
                                                    ├─ TGW Attachment ─ VPC-C
                                                    └─ ...（大量 VPC）
```

**優勢**：
- **Hub-and-Spoke 架構**：TGW 做「中央路由器」，集中管理路由與分段（TGW route tables）
- **擴展方式更乾淨**：新增 VPC = 新增 TGW attachment（不需要每個 VPC 都新增 VIF/VLAN/BGP）
- **Quota 壓力低**：只需要一條 Transit VIF，而不是線性增長
- **混合雲標準解**：一條（或少數）Transit VIF 對接大量 VPC，是現代企業網路的最佳實踐

**選擇標準**：
- **< 3 個 VPC**：Per-VPC Private VIF 還可以接受
- **≥ 3 個 VPC**：強烈建議用 Transit VIF + DXGW + TGW Hub

## 第五層：SiteLink

### 解決的問題

兩個 on-prem 辦公室想互通，不想另外租電信商的 WAN 專線

### 沒有 SiteLink vs 有 SiteLink

#### ❌ 沒有 SiteLink（要繞路，但還是私有）

```
台北 on-prem
   ↓ DX
AWS 骨幹
   ↓
某 Region 的 TGW（中繼）
   ↓
AWS 骨幹
   ↓ DX
高雄 on-prem
```

**問題**：延遲高、費用多（走 Region 中繼）

#### ✅ 有 SiteLink（直達，也是私有）

```
台北 on-prem
   ↓ DX Location A
AWS 骨幹直送
   ↓ DX Location B
高雄 on-prem
```

**優點**：延遲更低、費用更少（不需繞進 Region）

### 關鍵點

兩條路都不走公網，只是 SiteLink 在 AWS 骨幹內直連，不需中繼

### 啟用條件

✅ 兩端的 DX Connection 都要開啟 SiteLink
→ AWS 才能在骨幹上做直連路由

```
│                    AWS 私有骨幹                          │

│                                                         │
│  DX Edge Router                                         │
│       ↓                                                 │
│  VGW / TGW（Region 內的路由設備）                        │
│       ↓                                                 │
│  VPC（你在 AWS 上的私有網路，≈ 雲端版 LAN）               │
│                                                         │
│  SiteLink：讓兩個 DX Location 在骨幹內直連               │
└──────────────┬──────────────────────────────────────────┘

        DX Location
        （Colocation 機房，AWS 在這裡放 Router）

        你或 DX Partner 的設備
               │
        電信商 WAN 專線（不走公網）
               │
        你的辦公室 LAN
        （Totolink 這種 = 寬頻 = 走公網）
```

## 術語快速參考

### 基礎

- **LAN** = 自己地盤的網路，自己管
- **WAN** = 跨地點靠電信商的線
- **公網** = 全球 ISP 互連，大家共用
- **寬頻** = 共用公網出口，晚上會慢

### 企業連線

- **MPLS 專線** = 電信商給你保留的私有路徑，不走公網
- **Colocation** = 把你的硬體放在別人機房（停車位概念）

### AWS Direct Connect

- **DX Location** = AWS 指定的接入點機房
- **Cross-connect** = 機房內兩台設備之間的短光纖
- **Direct Connect** = 你拉進 AWS 私有骨幹的入口（實體線路）
- **AWS 骨幹** = AWS 自己的海底電纜+光纖，私有不共用

### AWS 路由

- **VGW / TGW** = AWS 骨幹上的路由設備，不走公網
- **VPC** = AWS 上的虛擬私有網路，概念上像雲端版 LAN
- **SiteLink** = 拿 AWS 骨幹當 WAN 用，兩個 on-prem 直連
