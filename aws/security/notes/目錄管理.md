# AWS 目錄服務（Directory Service）- SAP 考試備考

## Simple AD 三大考法與陷阱關鍵字

### 1. 成本與規模考量（The Low-Cost Choice）

當題目出現「需要一個低成本（Least expensive / Cost-effective）且受管的 Directory」時，Simple AD 通常是答案。

**關鍵場景：**
- 只有少量的使用者（Small 版本支援 500 使用者以下）
- 不需要複雑的 Windows 特性

**考點：** 如果題目強調預算有限，且只需要基本的「加入網域 (Domain Join)」功能，選 Simple AD。

---

### 2. 功能侷限性（The Compatibility Trap）

這是考試會考你「為什麼某個服務不能用」的題型。

**危險關鍵字（看到就排除 Simple AD）：**
- Microsoft Exchange
- Microsoft SharePoint
- Microsoft SQL Server（某些整合情境）
- Office 365

**考點：** 只要看到這些「重量級」微軟產品，Simple AD 絕對是錯誤選項。因為它是基於 Samba 4，而非真正的 Windows Server。

---

### 3. 信任關係（No Trust Relationships）

這是 Simple AD 在架構設計題中最常出現的「死穴」。

**考點：** Simple AD **不支援**與地端 AD 建立「信任關係 (Trust Relationship)」。

**考法對比：**

| 需求 | 推薦服務 |
|------|---------|
| 需要連接地端 AD，且保留地端帳號，且要建立 Forest Trust | AWS Managed Microsoft AD |
| 只需要一個雲端獨立的簡單目錄，不需連回地端 | Simple AD |
| 要把所有驗證請求「轉發」回地端（不存儲資料在雲端） | AD Connector |

---

## 快速選型對比表

| 需求特徵 | 推薦服務 | 考試關鍵字 |
|---------|---------|----------|
| 最便宜、基礎功能 | **Simple AD** | Samba 4, Small/Large, No Trust |
| 完整微軟相容性、信任關係 | **AWS Managed Microsoft AD** | Real Windows, Trust, MFA, High Availability |
| 純 Proxy、不存資料、地端驗證 | **AD Connector** | Proxy, On-premises sync, Direct Connect/VPN |
| 應用程式開發、非 AD 協定 | **Amazon Cognito / Cloud Directory** | Web/Mobile app, OAuth2, OIDC |

## 為什麼無法將 Exchange Server 與 Simple AD 連結？

### 核心考點：Simple AD 的相容性限制

**問題分析：** 雖然網路防火牆（Firewall）或 NACL 設定錯誤確實會導致連線失敗，但在架構設計的考題中，通常是在考「服務本質的限制」而非網路設定問題。

### Simple AD 的本質

Simple AD 是基於 **Samba 4** 開發的受管目錄服務，**並非真正的 Windows Server**。

**主要功能：** 提供基本的 Active Directory 功能（使用者帳戶、群組原則、加入網域）

**關鍵相容性限制：** 不支援許多需要進階 Active Directory 特性的 Microsoft 應用程式，包括：
- ❌ Microsoft Exchange
- ❌ Microsoft SharePoint
- ❌ Microsoft SQL Server（在某些整合情境下）
- ❌ Office 365 整合

### 選項拆解

| 選項 | 評價 | 理由 |
|------|------|------|
| A. Firewall 設定 | ❌ 網路問題，但非根本原因 | 通常會導致連線失敗，但若是相容性問題就無效 |
| B. NACLs 設定 | ❌ 網路問題，但非根本原因 | 同上 |
| **C. Simple AD 不支援 Exchange（正確）** | ✅ | 直接點出 Simple AD 的軟肋。Exchange 需要完整的 Active Directory Schema 支援與特定 API 調用，這些是基於 Samba 4 的 Simple AD 無法提供 |
| D. SSL 加密設定 | ❌ | LDAPS 是加密手段，並非導致無法連結的根本原因 |

## AWS 官方最佳實務建議

### 需要運行或連接 Microsoft Exchange 的解法

**方案 1：AWS Directory Service for Microsoft AD（Managed Microsoft AD）**
- 由真正的 Windows Server 運行的 AD
- 完整支援 Exchange、SharePoint 等企業應用
- 支援信任關係、MFA、高可用性

**方案 2：AD Connector**
- 若已有地端的 AD Server
- 作為 Proxy 將 AWS 服務的驗證請求導回地端 AD
- 也支援 Exchange

---

## 考場秒殺口訣

### 看到這幾個詞，請跳過 Simple AD

如果題目中看到以下需求，通常代表 Simple AD **不是**正確答案：

- ✋ "Establish a trust relationship with on-premises"（需要信任關係）
- ✋ "Schema extensions required for Exchange"（需要 Exchange 的 Schema 擴充）
- ✋ "More than 5,000 users"（使用者超過 5000 人）
- ✋ "MFA for local users"（Simple AD 的 MFA 支援較受限，通常考 Managed AD 才會配 MFA）

### 快速判斷流程

```
題目要求 → 連接地端 AD？
           ├─ YES → Managed Microsoft AD 或 AD Connector
           └─ NO → Simple AD 成本更低

           → 需要 Exchange/SharePoint/Office 365？
           ├─ YES → Managed Microsoft AD
           └─ NO → Simple AD 可行

           → 使用者超過 500？
           ├─ YES → 至少要 Large 版本或升級
           └─ NO → Simple AD Small 版本足夠
```
