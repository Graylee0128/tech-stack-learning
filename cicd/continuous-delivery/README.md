# Continuous Delivery 學習指南

這個目錄收的是「持續交付方法論」本身，不是特定工具教學。  
重點放在：

- 軟體交付為什麼會卡住
- `CI`、測試、部署流水線之間的關係
- config、環境、資料、相依性為什麼都屬於交付範圍
- 怎麼把交付從人治變成可重複的工程流程

## 目錄結構

### `notes/`

- `continuous-delivery-study-notes.md` - 以 `study-area.sre.tw` 的 `R2: Continuous Delivery` 為主軸重組的白話教學筆記

## 建議閱讀順序

1. 先看 `notes/continuous-delivery-study-notes.md`
2. 再回頭搭配 `cicd/github-actions/` 的工具型筆記
3. 最後把這些觀念對照到你自己的 build、test、deploy 流程

## 這個目錄在學什麼

如果 `github-actions/` 是「怎麼用某個 CI/CD 工具」，  
那 `continuous-delivery/` 比較像是「為什麼團隊需要這種交付能力，以及一條成熟交付鏈應該長什麼樣子」。
