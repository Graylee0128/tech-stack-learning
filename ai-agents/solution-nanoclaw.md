# NanoClaw 詳細分析

## 📌 專案概述

**NanoClaw** 是一個輕量級的開源個人 Claude 助手，採用 MIT 授權證，強調 **「一個進程，數個檔案，無複雜性」** 的設計理念。是本地部署最簡潔的方案。

**官方網站**: https://nanoclaw.net/
**GitHub**: zeroclaw-labs/nanoclaw
**授權**: MIT（完全開源）

## 🎯 核心功能

### 主要特性
1. **容器隔離安全** - 在 Linux 容器中運行代理，檔案系統隔離
2. **WhatsApp 整合** - 透過 WhatsApp 群組與 Claude 互動
3. **獨立群組記憶** - 每個 WhatsApp 群維持隔離的上下文記憶
4. **定時任務** - 內置任務排程器自動化日常操作
5. **多代理協作** - 支援多個 Claude 實例協調處理複雜任務
6. **完全本地化** - 除初次模型下載外，無雲服務依賴

### 架構特點
```
Single Process
    ↓
5 Core Files (輕量設計)
    ↓
Container Isolation (安全隔離)
    ↓
WhatsApp ←→ Claude Agent
```

## ⚠️ 潛在劣勢

1. **單一訊息平台** - 僅支援 WhatsApp（相比 OpenClaw 的 7+ 平台）
2. **Linux 容器依賴** - 主要針對 Linux，Windows/Mac 支援有限
3. **功能相對簡潔** - 無內置自動化任務編排（相比 OpenClaw）
4. **技能市場缺失** - 無預製技能生態（相比 OpenClaw 的 ClawHub）
5. **API 成本** - 仍需 Anthropic/OpenAI 付費 API

## 💾 部署與維護

### 安裝難度 - ⭐ 最簡單
```bash
# 典型安裝流程
1. 克隆倉庫或下載 5 個檔案
2. 設置 API 密鑰（ANTHROPIC_API_KEY）
3. 配置 WhatsApp 連線資訊
4. 運行容器
```

### 維護成本 - ⭐ 最低
- 最小的依賴清單
- 簡潔的除錯過程
- 容器隔離消除系統污染

### 系統需求
```
CPU:      低（可 ARM 架構）
RAM:      512MB ~ 2GB（足夠）
Storage:  1GB（包含模型）
Network:  僅需要出站 API 調用
```

## 🔧 配置與擴展

### 快速配置
- API 密鑰設置
- WhatsApp 群組綁定

### 代理配置方式

NanoClaw 基於 Anthropic Claude Agent SDK，使用標準的 SDK 配置方式來定義代理行為和能力。

💡 **注意**：NanoClaw 不原生支援 AIEOS 框架。如需 AIEOS 支援，請考慮 [ZeroClaw](./solution-zeroclaw-rust.md) 或 [OpenClaw](./solution-openclaw.md)。

### 擴展性
- 支援多代理執行個體
- 可透過修改 5 個核心檔案自訂邏輯
- AIEOS 框架支援代理人設自訂
- 社群貢獻的外掛/擴展（相對有限）
